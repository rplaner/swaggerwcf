Introdução

API de integração CoreCommerce

É possível integrar-se com o Core de várias maneiras, todas as operações estão disponíveis em nossa layer de integração e podem ser acessadas através dos seguintes protocolos:

  • REST

    • Você poderá realizar sua integração totalmente via os métodos padrões de POST/PUT/GET utilizando nossa WebAPI. A referência de todas as operações poderá ser vista aqui:
      [WEB API Reference]

  • SOAP

    • SOAP foi o primeiro método disponibilizado pela layer, os WSDLs das operações podem ser encontradas nos links abaixo:
      [Catalog]  [Configuration]  [Inventory]  [Payment]  [Pricing]  [Promotion]  [Queue]  [Sales]

Autenticação

Todas as requisições a API do CORE devem utilizar cabeçalhos do tipo BASIC AUTHORIZATION. Neste campo deve ser enviado a conta e senha de integração do cliente: 

Authorization: Basic Y29udGE6c2VuaGE=

O valor após a palavra Basic é uma chave Base64 da sua conta e senha. Para obter o valor, utilize o o comando base64 do linux: 

                                $ echo -n conta:senha | base64 
                                Y29udGE6c2VuaGE=

Ou através do site base64Encode. Lembrando que se já estiver logado podera testar a webapi e capturar as informações através de um software como o Fiddler.

Cenário Básico de Integração

Abaixo temos um esquema básico de integração com os métodos mais utilizados no CORE:

Consumindo um Item na Fila do CORE

O CORE possui o conceito de filas de integração, as quais servem para armazenar um tipo de objeto a ser consumido por um sistema externo. Podemos ter fila para todos os objetos do CORE, como Clientes, Pedidos, Produtos, etc. Geralmente o cliente irá configurar uma fila no Admin do CORE, designar um nome para ela e consumir os items da fila de tempo em tempo. Consumir um item da fila consiste em ler o item da fila, buscar as informações completas do item através do método apropriado e armazenar no sistema externo, por final depois deve-se acionar o método para retirar esse item da fila, marcando o mesmo como integrado.

Para ver quais filas seu sistema possui vá no ADMIN em:

Integrações => Gerenciamento de Filas

Após integrarmos a informação (nesse caso o pedido) no ERP devemos chamar o método para retirar o item consumido da fila, para não processarmos o mesmo repetidamente da próxima vez.

3 - Retirando o item consumido da fila :

Deverá ser chamado o método da API de QUEUE DequeueQueueItems com o devido corpo: (Notem que deve ser inserido o QUEUEITEMID do item retornado no primero passo de consulta a fila)

//POST: /v1/Queue/API.svc/web/DequeueQueueItems
{
  "QueueItems": [
    "d0a81f13-6072-4c02-a960-fec808983658"
  ]
}

Como Response teremos a confirmação de que o item saiu da fila:

{
  "IsValid": true,
  "Errors": [
    {}
  ]
}

(Os exemplos aqui colocados estão com suas pripriedades e métodos documentados inteiramente na seção de Referência).
